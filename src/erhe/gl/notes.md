# erhe_gl

## Purpose
Python-generated type-safe OpenGL API wrappers. Parses the Khronos `gl.xml` registry
to produce strongly-typed C++ enums for all GL enums, type-safe wrapper functions for
all GL entry points, call logging, error checking, dynamic function loading, and
extension queries.

## Key Types
- Generated enums in `wrapper_enums.hpp` -- Strongly-typed replacements for raw GL enum constants.
- Generated functions in `wrapper_functions.hpp` -- Type-safe wrappers around all GL functions.
- `gl_helpers` namespace -- Utility functions for format queries, size calculations, and format conversions between `erhe::dataformat::Format` and GL types.

## Public API
- All GL functions are available as `gl::function_name(...)` with strongly-typed parameters.
- `gl_helpers::size_of_type(type)` -- Get byte size of a GL type.
- `gl_helpers::convert_to_gl(format)` / `convert_from_gl(format)` -- Convert between `erhe::dataformat::Format` and `gl::Internal_format`.
- `gl_helpers::is_compressed()` / `is_integer()` / `has_depth()` etc. -- Query format properties.
- `gl_helpers::check_error()` / `set_error_checking(enable)` -- GL error checking.

## Dependencies
- **erhe libraries:** `erhe::dataformat` (public), `erhe::log` (private), `erhe::verify` (private)
- **External:** OpenGL (public), fmt (public)

## Notes
- Sources are generated by `generate_sources.py` from `gl.xml` and `gl_extra.xml` at CMake configure time.
- Generated files go to `${CMAKE_CURRENT_BINARY_DIR}/erhe_gl/` and include: `wrapper_enums.hpp`, `wrapper_functions.hpp/.cpp`, `dynamic_load.hpp/.cpp`, `enum_string_functions.hpp/.cpp`, `command_info.hpp/.cpp`, `enum_base_zero_functions.hpp/.cpp`.
- Templates in `templates/` directory control the code generation output.
- To regenerate, run `py -3 generate_sources.py <output_dir>` from the `src/erhe/gl/` directory.
